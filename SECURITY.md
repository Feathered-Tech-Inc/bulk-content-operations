# Security Policy

## Supported versions

Security fixes are applied to the latest released version and to the `main` branch.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Use one of the following private channels:

1. GitHub "Report a vulnerability" (preferred).
2. If private reporting is unavailable, contact repository maintainers directly through private project channels.

Include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Any suggested mitigation

## Response process

- We will acknowledge new reports as soon as practical.
- We will validate and triage severity.
- We will prepare and ship a fix, then publish coordinated disclosure notes.
- We may request additional details during investigation.
